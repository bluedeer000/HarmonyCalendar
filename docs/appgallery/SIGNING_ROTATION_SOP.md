# 签名轮换 SOP

## 目标

在旧签名材料疑似暴露后，完成安全轮换，并恢复 HarmonyOS 应用市场发布能力。

## 需要轮换的内容

- 发布证书
- 发布 profile
- keystore / `.p12`
- key alias 对应口令
- keystore store password

## 处理原则

- 旧签名材料不再继续用于后续版本发布。
- 新签名材料仅保留在本机安全位置或受控密码管理系统中。
- 仓库中只允许保留签名模板，不允许保留真实路径或真实口令。

## 轮换步骤

1. 在 DevEco Studio 或华为签名管理流程中重新申请/生成新的发布证书与 profile。
2. 导出新的 `.cer`、`.p7b`、`.p12` 以及对应口令。
3. 将旧签名材料从本机常用目录中隔离归档，避免误用。
4. 按 `LOCAL_SIGNING_SETUP.md` 在本机临时填入新的真实签名信息。
5. 执行仓库自检脚本，确认真实材料不会进入 Git。
6. 重新生成 signed HAP。
7. 用新签名完成真机安装验证。
8. 更新 `docs/appgallery/RELEASE_LOG.md`。

## 验证标准

- `build-profile.json5` 在 Git 中仍为模板版本
- 新 signed HAP 可安装
- 安装后应用可正常启动、同步、提醒
- AppGallery Connect 可识别新版本安装包

## 注意事项

- 如果华为应用市场对同一包名升级链依赖原签名，需要确认新签名是否影响既有升级路径。
- 如果旧签名已用于外部分发，轮换前应先评估升级兼容性。
- 如果需要保持原升级链，必须确认“泄露风险”与“升级连续性”的取舍后再操作。
